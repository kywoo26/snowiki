# Reference Structure

This directory holds durable supporting material that is **not** part of Snowiki's default canonical reading path.

## What belongs here

- explanatory architecture rationale beyond the two core contract docs
- research syntheses and comparison notes
- product-direction context
- external reference notes used by roadmap work
- isolated translation mirrors that should not compete with the canonical path

## Reading rule

Start with `docs/roadmap/` and the two key contract docs in `docs/architecture/` first.

Open `docs/reference/` only when you need supporting rationale, background evidence, or lineage context.
